Day-by-Day Build Workflow
Day 1 Backend Services
→ npm init + npm install express in each service folder
→ Write index.js with static JSON arrays for each service
→ Create Dockerfile per service (FROM node:20-alpine)
→ Write docker-compose.yml mapping ports 4001–4004
→ docker compose up -d → verify with curl /health
Day 2 Tenants, Roles & Users
→ Carbon Console → Configure → Multitenancy → Add finance.com
→ Add logistics.com same way
→ Login as financead@finance.com → create FinancePublisher + FinanceSubscriber roles
→ Create alice (publisher) + bob (subscriber) users
→ Assign Internal/publisher, Internal/creator, Internal/devops to alice
→ Repeat for logisticsad, carol, dave in logistics.com
Day 3 Publish APIs
→ Write 4 OpenAPI YAML files with OAuth2 security schemes
→ Publisher as alice → Create API → Import OpenAPI → invoice-api.yaml
→ Portal Configurations → Visibility → Restricted → FinanceSubscriber
→ Deployments → Deploy Revision → Lifecycle → Publish
→ Login as carol → publish ShipmentAPI + RouteAPI for logistics.com
→ Create API revisions on InvoiceAPI (v1.0-stable, v1.0-with-description)
Day 4 Dev Portal + OAuth2
→ Dev Portal ?tenant=finance.com → login as bob
→ Applications → Add FinanceApp → Generate Production Keys
→ Subscribe to InvoiceAPI + PaymentAPI
→ curl POST /oauth2/token with scope=invoice:read payment:read
→ Test scope enforcement → payment:read token hits invoice API → expect 900910
→ Repeat for dave + LogisticsApp in logistics.com
Day 5 Admin Portal
→ Admin → Rate Limiting → Subscription Policies → create FinanceGold, FinanceSilver, DevTest
→ Rate Limiting → Advanced Policies → create HeaderBasedThrottle
→ Rate Limiting → Deny Policies → create BlockTestIP (192.168.100.100)
→ Tasks → Workflow → test Application Creation approval
→ Publisher → InvoiceAPI → Monetization → Enable Monetization
Day 6 Mediation + CI/CD + Debug
→ Publisher → InvoiceAPI → Policies → add custom Synapse XML mediation
→ Add X-Tenant-ID header injection in request flow → verify in docker logs
→ Publisher Access Control → Restricted → FinancePublisher
→ Create userTheme.json → dark blue Dev Portal branding
→ apictl add env → apictl login → apictl export api InvoiceAPI
→ Add wire log config to log4j2.properties → test error codes
Day 7 Dashboard + Choreo Analytics
→ mkdir dashboard && npm install express cors node-fetch
→ Build server.js — WSO2 REST proxy + log parser + metrics aggregator
→ Add /_metrics endpoint to all 4 backend services → docker compose up -d --build
→ Wire dashboard to real health, APIs, logs, status codes, GitHub Actions
→ Configure Choreo: deployment.toml → [apim.analytics] config_endpoint + auth_token
→ Verify Choreo: 236+ calls in Insights dashboard, 60ms P95 confirmed
→ GITHUB_TOKEN=ghp_xxx node server.js — CI/CD page shows real run history
→ git add . && git commit && git push origin main
OAuth2 Token Flow
1
Client app — has Client ID + Secret from Dev Portal
2
POST /oauth2/token — grant_type=client_credentials + scope=invoice:read
3
WSO2 IS validates — checks client exists + subscription active
4
JWT issued — contains sub, scope, aud, exp claims
5
Client calls API — Authorization: Bearer <JWT>
6
Gateway validates — signature + expiry + scope + subscription
7
Backend called — X-Tenant-ID header added by mediation sequence
8
Response returned — JSON data from Node.js service

GitHub Actions CI/CD Pipeline — 5 Jobs
Job 1 Lint OpenAPI Specifications
→ npm install -g @stoplight/spectral-cli
→ spectral lint api-definitions/*.yaml --fail-severity error
→ Custom rules: api-must-have-description, operations-must-have-security
→ Duration: ~20s
Jobs 2+3 Deploy TenantA + TenantB (parallel)
→ apictl add env finance-local --apim https://localhost:9443
→ apictl login finance-local -u financead@finance.com -p FinAdmin@123 -k
→ apictl get apis -e finance-local -k (verify APIs exist)
→ apictl export api --name InvoiceAPI --version 1.0 --provider alice@finance.com
→ Duration: ~30s each (parallel execution)
Job 4 Post-Deploy Smoke Test
→ Get OAuth2 token for FinanceApp and LogisticsApp
→ curl -k https://localhost:8243/t/finance.com/invoices/1.0/invoices → expect 200
→ curl -k https://localhost:8243/t/logistics.com/shipmentapi/1.0/shipments → expect 200
→ All 4 gateway URLs verified — fails pipeline if any return non-200
Job 5 Slack Notification
→ Post success/failure summary to SLACK_WEBHOOK_URL secret
→ Includes run number, duration, status of each job
→ Duration: ~10s
